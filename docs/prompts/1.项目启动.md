**Role:** 你现在是一名资深 Python 游戏开发工程师，擅长编写逻辑严密、结构清晰的策略经营类游戏原型。

**Project Context:** 我们要开发一个关于“保罗旅行布道”的文字策略经营游戏。游戏的核心哲学是：**在既定的历史轨道（结局固定）中，通过管理有限资源完成使命。** 玩家不是在改变历史，而是在充满压力的环境中做一个忠心的管家。

**1. 核心数据结构设计**
请使用面向对象编程（OOP）构建以下基础框架：

- **Player 类：** 维护 4 个核心资源：
- `stamina` (体力)：初始 100。
- `provision` (物资)：初始 100。
- `stability` (教会健康度)：初始 0。
- `persecution` (逼迫指数)：初始 0。

- **City 类：** 包含名称、基础逼迫增长率、固定回合数。
- **GameEngine 类：** 负责回合流转、动作处理及数值结算。

**2. 行动系统逻辑 (Action Mapping)**
请实现一个基础的 `handle_action` 函数，支持以下操作及数值修正：

- **公开讲道：** 体力 -20, 教会健康 +15, 逼迫指数 +20。
- **织帐棚：** 体力 -15, 物资 +30, 教会健康 -2。
- **私下门训：** 体力 -10, 教会健康 +5, 逼迫指数 +2。
- **休息：** 体力 +20, 物资 -10。

**3. 城市流转逻辑**
MVP 版本包含三个城市：**安提阿 (Antioch)** -> **腓立比 (Philippi)** -> **以弗所 (Ephesus)**。

- 每个城市固定为 3 个回合。
- 回合结束时，检查 `persecution > 70` 是否触发“暴动事件”，或 `stability < 30` 是否触发“分裂风险”。

**4. 开发要求**

- 请先写出核心的 Python 代码框架（不带 GUI，纯 Terminal 输出）。
- 代码需具备良好的注释，方便后续扩展“同工系统”和“书信系统”。
- **输出要求：** 先展示代码结构，然后提供一个可直接运行的 `main.py` 原型。
